# AI Product & Technical Program Portfolio
Portfolio samples for evaluation/hiring purposes.

This portfolio showcases AI product, technical program management, and data-platform work across agentic workflows, RAG evaluation, media analytics, and streaming ads measurement.

The projects are designed from a product and technical program leadership perspective: problem framing, architecture, workflow design, user stories, evaluation criteria, KPIs, launch risks, and implementation-ready artifacts.

## Portfolio Purpose

This portfolio demonstrates my ability to translate ambiguous business and technical problems into structured AI-enabled workflows, product requirements, delivery plans, evaluation frameworks, and measurable outcomes.

The focus is not only on building AI prototypes, but on showing how AI systems should be designed, evaluated, governed, and launched responsibly in enterprise environments.

## Projects

### 1. Enterprise AI Workflow Agent

An agentic workflow prototype that converts ambiguous business requests into structured product and delivery artifacts.

Outputs include:

- User stories
- Acceptance criteria
- Risks and dependencies
- KPIs
- Launch checklist
- Human-review flag for low-confidence or high-risk outputs

This project is most relevant to AI product management, technical program management, product operations, workflow automation, and enterprise AI adoption roles.

### 2. RAG Evaluation Dashboard

A lightweight evaluation framework for AI assistant and search responses.

The project evaluates responses across:

- Groundedness
- Context relevance
- Answer relevance
- Citation quality
- Hallucination risk
- Escalation need

This project is most relevant to AI search, RAG systems, AI assistant quality, evals, product analytics, and responsible AI launch readiness.

### 3. Streaming Ads Measurement Product Case Study

Artifacts include:

- Event schema

- Product requirements
- Rollout plan
- Risk register

This project is most relevant to media technology, streaming platforms, ad tech, measurement, data products, and analytics product management roles.

## How to Review This Portfolio

Each project folder includes a project-specific README and supporting artifacts.

Recommended review order:

1. Start with the project README.
2. Review the architecture or product requirements document.
3. Review the sample input/output or evaluation rubric.
4. Review the launch, risk, or KPI framework.

The goal is to show how I think through AI-enabled products from idea to execution, not just isolated code snippets.
